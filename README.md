## 1. ATProto 中没有实例

**原文标题**: There Are No Instances in ATProto

**原文链接**: [https://overreacted.io/there-are-no-instances-in-atproto/](https://overreacted.io/there-are-no-instances-in-atproto/)

《ATProto 中没有“实例”》一文指出，人们普遍寻找“Bluesky 实例”的行为是一种范畴错误，源于对去中心化协议的一种“Mastodon 式思维”误解。

作者解释说，Mastodon (ActivityPub) 的运作方式类似于由联邦化的“微型 Facebook”（即实例）组成的网络，其托管与应用是捆绑在一起的；而 ATProto 则采用了类似于早期 Web 中 RSS 与 Google Reader 的模式。在这种模式下，**托管与聚合是解耦的。**

**文中强调的核心区别包括：**

*   **身份可迁移性：** 在 Mastodon 中，你的身份绑定在特定实例上（例如 @user@instance.com）。如果该实例消失或“断开联盟”，你的身份和社交图谱就会受损。在 ATProto 中，你的身份独立于托管方；你可以在不同服务商之间迁移数据（或自建托管），而不会丢失账号或关注者。
*   **RSS 类比：** 作者将 ATProto 比作“博客圈”。博客（托管）独立存在，而像 Google Reader 这样的应用（聚合）只是对这些数据进行投影。ATProto 应用是整个网络数据的投影，而非孤立的信息孤岛。
*   **网络拓扑：** Mastodon 依赖于各“领地”之间复杂的联盟协议。ATProto 则允许应用同时从所有人的托管中聚合内容，避免了将数据存储与用户界面耦合这一“原罪”。

最后，作者总结道，衡量 ATProto 去中心化程度的标准不应是“实例”的数量，而应是独立托管服务商的多样性，以及用户选择用来浏览同一共享数据的各类应用（如 Tangled 或 Semble）的多样性。将托管与应用分离，赋予了用户真正的自主权，并修复了中心化社交媒体中扭曲的激励机制。

## 3. 业余人士或已破解困扰120年的线形文字A之谜

**原文标题**: Amateur may have cracked Linear A, a 120-year-old puzzle

**原文链接**: [https://aiclambake.com/clamtakes/linear-a/](https://aiclambake.com/clamtakes/linear-a/)

自学成才的人工智能工程师兼业余语言学家汤姆·迪·米诺（Tom Di Mino）声称，他已成功破译了线形文字A。这是一种青铜时代的米诺斯书写系统，120多年来一直是一个未解之谜。目前，罗格斯大学和剑桥大学的专家正在对他的研究成果进行评审。其研究表明，线形文字A对应着一种已灭绝的闪米特语，是圣经希伯来语的前身。

尽管相关的线形文字B已于1952年被破译为希腊语，但线形文字A虽然与前者共有60个符号，却始终难以攻克。迪·米诺的突破发生在2024年5月，当时他正在分析一段公式化的祈祷铭文。通过识别出意为“居住”的三辅音词根N-W-Y，他成功破解了一个反复出现的动词含义，并随后推导出更广泛的文本内容。他的研究指出，这些古老的祈祷词是献给一位女神的，且在结构上与后来的希伯来连祷文具有相似性。

迪·米诺方法的一个关键区别在于人工智能的应用。他利用“Claude Code”构建了一个系统化引擎，用于整理符号、测试假设并对数据进行分类。这种方法使他能够：
* 提出37个字符的读法，其中包括13个线形文字A特有的符号。
* 解决了线形文字B中三个此前未知的音值。
* 开发出一份包含383个已翻译米诺斯词项的词典。

如果得到证实，迪·米诺的研究将代表语言学领域的重大变革，可与线形文字B的破译相媲美。他已经撰写了一份名为《Ya Diktu：米诺斯山顶圣所祭酒公式语法》（*Ya Diktu: Grammar of the Minoan Peak Sanctuary Libation Formula*）的手稿草案，准备提交同行评审发表。除了眼前的翻译成果，迪·米诺认为这种由AI驱动的技术可以为破解其他尚未破译的古代书写系统提供路线图。

## 4. Project Valhalla 详解：十年研发成果如何落地 JDK 28

**原文标题**: Project Valhalla, Explained: How a Decade of Work Arrives in JDK 28

**原文链接**: [https://www.jvm-weekly.com/p/project-valhalla-explained-how-a](https://www.jvm-weekly.com/p/project-valhalla-explained-how-a)

Valhalla 项目是一项旨在现代化 Java 内存模型的十年计划。该项目已正式通过 **JEP 401（数值类与对象）** 瞄准 **JDK 28**。作为一项预览特性，它旨在实现一个长久以来的目标：“编写如类，运行如数”。

**问题所在**
自 1995 年以来，Java 在原生类型（快速、紧凑）与对象（缓慢、冗余）之间保持着严格的界限。对象需要头部信息和指针跳转，这导致了较差的“引用局部性”。现代 CPU 的速度远快于内存，因此这些“指针跳转”会导致缓存未命中，从而降低性能。

**解决方案：数值类**
Valhalla 引入了通过 `value` 修饰符声明的**数值类 (Value Classes)**。其定义性特征是**缺乏标识 (Identity)**。
*   **无标识：** 两个字段完全相同的数值对象被视为可替换的。`==` 运算符比较的是它们的内部状态，而非内存地址。
*   **约束条件：** 数值类隐式为 final，其字段也必须为 final，且不能用于同步锁（否则会抛出 `IdentityException`）。
*   **性能提升：** 通过消除标识，JVM 可以执行**标量化 (Scalarization)**。JIT 编译器会将对象拆解为独立的字段，并将其存储在寄存器或栈中，从而完全避免堆分配。

**关键细节与未来工作**
虽然这是一个重要的里程碑，但 JDK 28 的发布仅是 Valhalla 的“第一部分”：
*   **可空性：** 在 JDK 28 中，数值对象仍是引用类型，可以为 `null`。要实现完全的“堆扁平化”（像原生类型一样将对象紧凑地存储在数组中），需要非空类型，这将在未来的 JEP（空受限数值类类型）中实现。
*   **泛型：** 通用/特化泛型（将允许 `ArrayList<int>`）仍属于该项目未来的独立阶段。

总之，JDK 28 标志着一场根本性变革的开始，它允许开发者编写安全、可读的抽象，同时让 JVM 能够以硬件原生类型的效率对其进行优化。

## 5. 新法案针对旨在压制合法网络言论的政府施压行为

**原文标题**: A new bill takes aim at government pressure to silence lawful online speech

**原文链接**: [https://www.eff.org/deeplinks/2026/06/new-bill-takes-aim-government-pressure-silence-lawful-online-speech](https://www.eff.org/deeplinks/2026/06/new-bill-takes-aim-government-pressure-silence-lawful-online-speech)

参议员特德·克鲁兹（Ted Cruz）和罗恩·怀登（Ron Wyden）提出了《针对网络表达中官僚机构权力过度扩张的正义法案》（Justice Against Weaponized Bureaucratic Overreach to Networked Expression，简称 JAWBONE 法案）。这项跨党派立法旨在打击“施压行为”（jawboning），即政府官员向社交媒体平台、广播机构和人工智能提供商等私人中介机构施压，以审查受第一修正案保护的合法言论的做法。

该法案建立了两种主要的问责机制：
1. **法律追究**：它创立了一项联邦诉讼理由，允许个人起诉那些强迫平台压制受保护表达的政府官员。
2. **透明度**：它强制建立一套制度，要求披露政府与科技公司之间关于用户内容的沟通记录。

电子前哨基金会（EFF）对该法案表示支持，并提及其代表“ICEBlock”应用开发者约书亚·亚伦（Joshua Aaron）发起的诉讼。据称，在美国总检察长及联邦官员威胁要进行调查和起诉后，亚伦的应用被苹果应用商店下架。EFF 目前正通过《信息自由法》（FOIA）诉讼，以揭露涉及谷歌（Google）和 Meta 的类似强制性沟通。

尽管 EFF 主张终止此类违宪压力，但该组织也强调应采取审慎的态度。它指出，并非所有政府与平台之间的沟通是有害的，合法的共享信息对于公共安全至关重要。此外，EFF 强调私人平台并非“国家行为体”。根据宪法第一修正案，这些公司保留其编辑权，可独立于政府意愿自行策划和管理内容。

最终，JAWBONE 法案旨在寻求一种平衡，在保护普通用户言论自由的同时，确保政府无法在幕后操控在线话语权。
