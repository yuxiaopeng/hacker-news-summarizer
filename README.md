## 2. 伦敦—加尔各答巴士服务

**原文标题**: London–Calcutta Bus Service

**原文链接**: [https://en.wikipedia.org/wiki/London%E2%80%93Calcutta_bus_service](https://en.wikipedia.org/wiki/London%E2%80%93Calcutta_bus_service)

伦敦—加尔各答大巴服务是一条运行于1957年至1976年间的历史性长途国际线路。它被广泛认为是世界上最长的巴士线路，单程跨越约1万英里，耗时约50天完成。

**线路与体验**
该服务最初由艾伯特旅游公司（Albert Travel）推出，于1957年4月15日开启首航。线路横跨欧洲和中东，途经比利时、西德、南斯拉夫、土耳其、伊朗、阿富汗和巴基斯坦等国。进入印度后，大巴经由新德里和阿格拉抵达加尔各答。该服务提供全包式体验，涵盖交通、餐饮和住宿。在当时，这辆巴士的配置极其先进，配备了卧铺、暖风机、厨房以及上层观景休息室。乘客还可以沿途游览泰姬陵等名胜，并在萨尔茨堡和伊斯坦布尔等城市购物。

**后期发展与扩张**
1968年，英国旅行者安迪·斯图尔特（Andy Stewart）购买并改装了这辆巴士，将其命名为“艾伯特号”（Albert）。在艾伯特旅游（Albert Tours）的运营下，该服务得到了扩展，新增了一条经由加尔各答、新加坡并跨海抵达澳大利亚，连接伦敦与悉尼的线路。这段延伸后的旅程耗时约132天。

**停运**
在完成约15次往返航程后，该服务于1976年永久停运。其停运归因于日益动荡的地缘政治局势，包括引发伊朗革命的紧张局势以及印巴之间不断升级的冲突，这使得这条陆上“嬉皮士之路”因过于危险而无法进行常规运输。在1973年的鼎盛时期，一张从伦敦到加尔各答的车票售价为145英镑。

## 4. 猫砂引发的核灾难

**原文标题**: When Kitty Litter Caused a Nuclear Catastrophe

**原文链接**: [https://practical.engineering/blog/2025/4/15/when-kitty-litter-caused-a-nuclear-catastrophe](https://practical.engineering/blog/2025/4/15/when-kitty-litter-caused-a-nuclear-catastrophe)

2014年2月，位于新墨西哥州的废弃物隔离试点工厂（WIPP）——一个旨在将核废料封存在深层地下盐穴中的处置库——发生了放射性物质泄漏。经追溯，泄漏源自洛斯阿拉莫斯国家实验室（LANL）的一个废料桶，该桶发生了爆裂并起火。

调查揭示了一个离奇的起因：一起涉及猫砂的文书错误。为了稳定硝酸盐液体废料，技术人员本应添加沸石粘土等无机吸收剂。然而，在一次操作规范更新中的沟通脱节，导致“无机”一词被误写成了“有机”。结果，承包商使用了名为“Swheat Scoop”的小麦基有机猫砂。这种有机材料与硝酸盐发生了化学反应，引发热失控，导致废料桶内压力过大并最终破裂。

尽管WIPP的安全系统和高效空气过滤器（HEPA）成功减轻了对环境的影响，但仍有21名工人暴露于低剂量辐射中。该设施被迫关闭了近三年，清理及更换通风系统的总费用约5亿美元。

这一事件为管理那些在数千年内仍具有危险性的有害物质敲响了警钟。它凸显出，即便是像WIPP地质结构稳定的盐穴这样最先进的工程方案，在人为错误和系统性监管疏漏面前也显得十分脆弱。最终，这场“猫”祸强调了在处理对人类构成永久威胁的物质时，保持极度警惕和精确沟通的必要性。

## 5. Linux 成功运行于树莓派 RP2350 的 Hazard3 RISC-V 内核 (2024)

**原文标题**: Linux Runs on Raspberry Pi RP2350's Hazard3 RISC-V Cores (2024)

**原文链接**: [https://www.hackster.io/news/jesse-taube-gets-linux-up-and-running-on-the-raspberry-pi-rp2350-s-hazard3-risc-v-cores-19088b87bb2c](https://www.hackster.io/news/jesse-taube-gets-linux-up-and-running-on-the-raspberry-pi-rp2350-s-hazard3-risc-v-cores-19088b87bb2c)

开发者 Jesse Taube 已成功在树莓派 RP2350 微控制器上启动了一个极简 Linux 发行版，利用了其集成的开源 Hazard3 RISC-V 内核。这一里程碑式的成果展示了在低成本微控制器上运行通常仅限于应用级处理器软件的能力。

虽然 RP2350 配备了双 Arm Cortex-M33 内核，但 Taube 的工作聚焦于芯片上的两个 Hazard3 RISC-V 内核，代码在这些内核上原生运行。相比需要通过模拟才能运行类似软件的旧款 RP2040，这是一个重大的飞跃。

实现这一成果主要面临两个技术限制：
1. **内存管理**：由于 RP2350 缺乏内存管理单元 (MMU)，它需要特定版本的 “NOMMU” Linux 内核。
2. **内存容量**：微控制器内部的 520kB SRAM 不足以承载 Linux。为了运行，系统需要外部伪静态随机存取存储器 (PSRAM)。

由于上述要求，该 Linux 发行版无法在缺乏 PSRAM 扩展的标准 Raspberry Pi Pico 2 上运行。然而，它已在 SparkFun Pro Micro RP2350 上成功验证，这是一款集成了 8MB PSRAM 的第三方开发板。

Taube 已在 GitHub 上发布了 Buildroot 配置和设置指南，方便其他开发者在 RP2350 平台上尝试运行 Linux。
