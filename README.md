## 2. 售出2500台MIDI录音机的感悟：硬件其实没那么难

**原文标题**: What I learned selling 2,500 MIDI recorders: Hardware is not so hard

**原文链接**: [https://chipweinberger.com/articles/20260719-hardware-is-not-so-hard](https://chipweinberger.com/articles/20260719-hardware-is-not-so-hard)

在本文中，自动钢琴 MIDI 录音机 **Jamcorder** 的开发者回顾了成功售出 2,500 台产品的历程，并对“硬件很难”这一传统观念提出了挑战。

作者认为，对于像 Jamcorder 这样的分众市场产品，软件开发的过程远比硬件更为艰巨。固件和应用程序耗费了 20 万行代码和三年的心血，而硬件生产则“一帆风顺”。作者甚至在短短四天内就亲手组装了前 500 台设备，并未遇到预想中的制造噩梦。

成功的关键在于**刻意的简约**。作者通过以下方式确保了硬件的可控性：
*   使用简单的 PCB，仅包含 25 种独特的、且大多是现成的组件。
*   采用易于组装的设计（仅需一颗螺丝，无需复杂的注塑模具滑块结构）。
*   果断舍弃非必要功能，如 USB-C、电源按钮和电池检测。

作者坦言，虽然在超大规模生产或低利润的竞争市场（如智能手机）中硬件确实很难，但它通常“取决于你把它做得多难”。对于独立开发者而言，只要能控制复杂度，硬件开发是一条可行的道路。

**中等规模硬件出货的核心建议包括：**
*   **保持高利润率：** 目标毛利率应至少达到 70%。
*   **简化物料清单 (BOM)：** 避免使用单一供应商提供的组件和复杂的校准流程。
*   **战略性采购：** 通过阿里巴巴等平台与中国组装厂合作，但将最终的质量保证 (QA) 和库存留在本地。
*   **运营规范：** 使用图片记录制造步骤，制定强有力的防伪策略，并尽量缩小包装以降低成本。

## 4. 英国雷利花园香蕉在15年后开花结果

**原文标题**: Bananas sprout in Rayleigh Garden UK after 15 years

**原文链接**: [https://www.bbc.com/news/articles/cvg8edqq5g5o](https://www.bbc.com/news/articles/cvg8edqq5g5o)

经过多年的栽培，英国埃塞克斯郡雷利和萨福克郡佩蒂斯特里的花园中，香蕉树已成功结果。种植了200棵香蕉树的彼得·斯塔夫和艾玛·斯塔夫夫妇报告称，这些植株在15年后首次结果。他们将这一里程碑式的成果归功于近期的热浪以及在花园中营造出的集热微气候。

专家认为，这一现象是气候变化的直接结果。皇家园艺学会首席园艺师盖伊·巴特指出，随着英国夏季愈发炎热、冬季愈发温和，橄榄、无花果和杏等“喜热”植物正蓬勃生长。相比之下，醋栗和大黄等传统英国作物在某些地区已开始难以生长。

虽然芭蕉（Musa Basjoo，又称耐寒香蕉）以能在英国的寒冬中生存而闻名，但其花朵通常需要防霜处理才能存活。植物学家詹姆斯·黄解释说，尽管在当前气候下，这类植物“生长速度极快”且日益易于养护，但其果实基本无法食用，且布满种子。

这一趋势凸显了英国园艺业的重大转变：气温上升使得外来物种能够在以前无法结果的地区繁茂生长。

## 5. Claude Code 现已使用 Rust 编写的 Bun

**原文标题**: Claude Code uses Bun written in Rust now

**原文链接**: [https://simonwillison.net/2026/Jul/19/claude-code-in-bun-in-rust/](https://simonwillison.net/2026/Jul/19/claude-code-in-bun-in-rust/)

本文报告称，截至 2026 年 7 月，**Claude Code**（v2.1.181 及更高版本）已转换至使用由 **Rust** 重写的 **Bun** 运行时版本。

据 Bun 的创始人 Jarred Sumner 称，迁移到 Rust 版本使 Linux 上的启动速度提升了 10%。此次转换被描述为“平淡无奇”（boring），意指其过程非常无缝，以至于大多数用户并未察觉到这一重大的架构变化。

作者通过检查 `claude` 二进制文件验证了这些说法，并发现了两个主要证据：
1. **未发布的版本号：** 该二进制文件标识为 Bun v1.4.0，这是一个在撰写本文时领先于官方稳定版的预览版本。
2. **Rust 源代码：** 通过使用 `strings` 命令，作者在二进制文件中发现了 500 多处对 `.rs`（Rust）文件路径的引用（例如 `src/bundler/bundle_v2.rs`），从而确认了底层语言的转变。

文章最后指出，Rust 版本的 Bun 已经运行在数百万台设备的生产环境中。虽然尚未进入通用稳定通道，但该基于 Rust 的版本目前已通过 Bun 的 canary（金丝雀）版本向公众开放。

## 6. Minecraft：Java版现已使用SDL3

**原文标题**: Minecraft: Java Edition now uses SDL3

**原文链接**: [https://www.minecraft.net/en-us/article/minecraft-26-3-snapshot-4](https://www.minecraft.net/en-us/article/minecraft-26-3-snapshot-4)

Minecraft 快照 26.3 Snapshot 4 引入了一项重大的技术性大修，最显著的变化是将游戏的窗口管理和输入后端从 **GLFW 切换到了 SDL3**。这一变化为 Linux 带来了原生 Wayland 支持，改进了 macOS 上的文本输入，并转向使用物理按键绑定（扫描码）。无边框全屏现在成为了默认模式，而“原始输入”在游戏过程中已被永久性的相对鼠标模式所取代。

**UI 与游戏玩法调整**
调试信息界面 (F3) 现在支持自定义 GUI 缩放、玩家速度追踪和刷新率显示。创造模式物品栏经过了重新组织，使矿物和自然方块的排列逻辑更符合直觉。玩法调整包括允许旁观者使用传送门，并防止犰狳在浸入液体时卷缩。

**技术与数据改动（数据包 111.0）**
此快照引入了几个强大的新数据组件：
*   **燃料：** `cooking_fuel` 和 `brewing_fuel` 允许自定义燃烧时间和速度倍率。
*   **工具：** `villager_food`、`waxed`（涂蜡）标记和 `mob_visibility`（用于修改生物探测范围）。
*   **告示牌：** 为了增强安全性，告示牌默认不再执行点击事件或解析文本组件；现在需要新的 `allow_op_features` 标志来恢复此功能。

此外，酿造台和熔炉现在使用整数 (integers) 而非短整数 (shorts) 来存储时间和燃料数据，从而支持更长的持续时间。

**注册表与进度**
战利品表、谓词和进度的技术后端已完成现代化改造。大多数字段现在支持灵活的“引用”系统，允许创作者交替使用内联值、命名空间 ID 或带井号前缀的标签。许多进度触发器也已更新，支持直接引用谓词，从而简化了复杂数据包的制作流程。

## 7. Blender 5.2 长期支持版

**原文标题**: Blender 5.2 LTS

**原文链接**: [https://www.blender.org/download/releases/5-2/](https://www.blender.org/download/releases/5-2/)

Blender 5.2 LTS 于 2026 年 7 月 14 日发布，为程序化工作流、物理模拟和渲染性能带来了变革性的更新。

**几何节点与物理**
一大亮点是引入了由全新 **XPBD 解算器**驱动的节点式物理系统。这实现了高度可定制的程序化布料和毛发模拟。几何节点还通过 **Sample Sound Frequencies**（音频频率采样）节点获得了音频反应能力，并新增了备受期待的 **Mesh Bevel**（网格倒角）节点，以及用于高级数据管理和传输的 “Lists”（列表）和 “Bundles”（捆绑包）。

**渲染：Cycles 与 EEVEE**
Cycles 引入了全新的**纹理缓存（Texture Cache）**系统，通过生成优化的 `.tx` 文件显著降低了内存占用。Cycles 和 EEVEE 现在的原理化 BSDF 均支持 **Thin Wall（薄壁）**模式，能够真实地渲染叶片和纸张等半透明材料。EEVEE 经历了重大重构，在实例密集的场景中速度提升了 2 倍，通过“背面”屏幕追踪改进了全局光照，并增强了与 Cycles 的一致性。颜色管理也得到了扩展，新增了多种行业标准的摄影机色彩空间。

**资产与合成器**
资产系统转为**“按需加载（On Demand）”**模式；Essentials 基础库现托管在云端，以减小软件体积，用户也可以托管自己的在线仓库。合成器迎来大规模扩展，新增了 6 种接口类型（包括矩阵和字符串）以及 35 个新节点。全新的**合成资产（Compositing Assets）**为水彩或胶片等风格提供了“即时外观（Instant Looks）”，同时 Gizmo 工具现在支持自动打帧，并在图像编辑器中提供了更好的对齐支持。

**性能与工具**
常规改进包括更快的属性求值、增强的字符串处理能力，以及将几何节点应用于空对象的能力。此版本专注于“惰性求值”和去重技术，以确保所有模块都能提供更流畅的用户体验。
