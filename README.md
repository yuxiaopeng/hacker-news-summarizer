**原文标题**: The C3 Programming Language

**原文链接**: [https://c3-lang.org](https://c3-lang.org)

C3 是一种编程语言，定位为 C 语言在人体工程学、安全性和熟悉度上的进化版本。它的目标是在改进 C 语言基础的同时，为资深 C 开发者保留熟悉的语法和工作流。

**主要特性与亮点：**
*   **设计理念：** 该语言专注于“人体工程学”和“安全性”，旨在比传统 C 语言进一步提升开发效率和内存安全性。
*   **跨平台支持：** C3 可在 Windows、MacOS 和 Linux 等主流操作系统上下载使用。
*   **现代语法：** 该语言采用基于模块的系统（例如 `module hello_world;`）和结构化的标准库（例如 `import std::io;`）。
*   **熟悉感：** 尽管进行了更新，该语言仍保留了类似于 C 的结构，使用 `main` 函数以及像 `io::printn` 这样直观的命令调用。

总之，C3 是 C 语言的现代化演进，力求在不放弃其前身特有的效率和逻辑的前提下，提供一个更安全、更易用的环境。

## 2. 心血管疾病是一个已解决的问题。

**原文标题**: Cardiovascular disease is a solved problem

**原文链接**: [https://totalhealthoptimization.com/2025/09/27/cardiovascular-disease-is-a-solved-problem/](https://totalhealthoptimization.com/2025/09/27/cardiovascular-disease-is-a-solved-problem/)

本文认为，鉴于现代药理学的进步以及对其生物学病因的清晰认识，心血管疾病（CVD）这一全球首位死因实际上已成为一个“已解决的问题”。

作者指出，心血管疾病的根本原因在于低密度脂蛋白（LDL）——特别是含有载脂蛋白 B (apoB) 和脂蛋白(a) [Lp(a)] 的颗粒——在动脉壁内的终身累积。这些颗粒会引发适应不良的炎症反应，从而导致动脉粥样硬化斑块。关键在于，作者认为心血管疾病风险取决于对这些颗粒的**累计终身暴露**，而非短期水平；这意味着即使在 LDL 水平被视为“正常”时，损害也往往在青少年时期就已经开始。

文章重点列举了两大类证据，用以证明 LDL 是心血管疾病的直接原因，而不仅仅是相关因素：
1. **遗传证据**：孟德尔随机化研究显示，具有较低 LDL 遗传倾向的个体，其患心脏病的终身风险显著降低。
2. **临床试验**：数十年关于他汀类药物和现代生物制剂（如 PCSK9 抑制剂）的数据一致表明，降低 LDL 可降低心血管死亡率。

此外，作者阐明，与传统的 LDL 质量测量（mg/dL）相比，**apoB 颗粒计数**是预测风险更优的指标。

作者总结道，由于我们拥有安全且耐受良好的药理学工具来切断动脉粥样硬化的因果路径，心血管疾病是完全可以预防的。对于健康的年轻人来说，积极且早期地降低 LDL 代表了一份“免费午餐”——这是一种低风险的干预措施，几乎可以在人的整个生命周期中消除心血管疾病的发生。文章呼吁向真正的预防医学转变，以在后代中根除这种疾病。

## 4. 在自有网站发布，在其他平台分发。

**原文标题**: Publish on your own site, syndicate elsewhere

**原文链接**: [https://indieweb.org/POSSE#](https://indieweb.org/POSSE#)

**POSSE** 是 **Publish (on your) Own Site, Syndicate Elsewhere**（在自有网站发布，同步至他处）的缩写。它是 IndieWeb 的核心理念，鼓励创作者在将内容分享到第三方社交媒体“孤岛”之前，先将其托管在个人域名上。通过先在个人网站发布，随后将副本或链接分发到 Twitter、Facebook 或 Mastodon 等平台，用户在保持数据首要所有权的同时，仍能触达使用这些服务的友邻。

本文概述了这一方法的核心优势：
*   **所有权与独立性：** 减少对第三方服务的依赖，确保即使某个“孤岛”关停或更改服务条款，内容依然可以访问。
*   **规范标识：** 原始 URL 保留在用户的域名下，为内容提供永久的归宿。
*   **社交触达：** 优先考虑人际关系，让用户在不被锁定在特定平台生态系统的情况下，依然能与现有平台上的朋友保持联系。
*   **增强发现性：** 在同步发布的帖子中包含“永久短链接”，有助于搜索引擎提高原创内容的排名，并引导用户发现创作者的独立站点。

实现过程通常遵循“从网站到孤岛”的工作流。当用户在自己的服务器上发布内容时，软件会自动（或半手动）将版本推送到选定的社交网络。文中提到了多种实现工具，如 **Bridgy Publish**、**IFTTT** 以及针对 WordPress 等 CMS 平台的专用插件。

最终，POSSE 代表了从数字单一文化向去中心化网络的转型。它在保留社交网络优势（如信息聚合和社交层）的同时，确保由个人而非企业保留其数字生活的“规范”副本。

**原文标题**: Daft Punk Easter Egg in the BPM Tempo of Harder, Better, Faster, Stronger?

**原文链接**: [https://www.madebywindmill.com/tempi/blog/hbfs-bpm/](https://www.madebywindmill.com/tempi/blog/hbfs-bpm/)

在《<Harder, Better, Faster, Stronger>的BPM节奏中藏有Daft Punk的彩蛋？》一文中，John Scalo指出，这首经典曲目的节奏并非大多数音乐数据库中所标注的标准123 BPM，而恰好是**123.45 BPM**。

作为一名节奏检测应用的开发者，Scalo注意到其软件始终将这首歌识别为带有小数的节奏。为了验证这一点，他对高品质CD转录音源进行了手动波形分析。通过测量首尾拍点之间的精确时长，并代入公式 $bpm = 60 \times (\text{拍数} - 1) / \text{时长}$，他计算出的节奏为 **123.4499**——与123.45几乎完美契合。

作者还调查了Daft Punk当时的设备（约1999-2000年间）是否支持如此高的精度。虽然像Akai MPC-3000这样的硬件采样器仅支持一位小数，但存档访谈证实，这对组合当时也使用了**Emagic的Logic软件**，该软件支持精确到小数点后四位的BPM设置。

Scalo总结道，“1-2-3-4-5”这一数字序列很可能是“机器人”们有意为之的顽皮选择。尽管Thomas Bangalter后来声称该组合在早期专辑中并未使用电脑，但技术证据表明，这一特定节奏是一个被刻意隐藏在眼皮底下近25年的彩蛋。

## 7. 递归语言模型

**原文标题**: Recursive Language Models

**原文链接**: [https://arxiv.org/abs/2512.24601](https://arxiv.org/abs/2512.24601)

**Summary of "Recursive Language Models"**

In the paper "Recursive Language Models," authors Alex L. Zhang, Tim Kraska, and Omar Khattab introduce **Recursive Language Models (RLMs)**, a novel inference strategy designed to overcome the limitations of fixed context windows in large language models (LLMs).

The core innovation of RLMs is treating an arbitrarily long prompt as an external environment rather than a static input. Instead of attempting to process the entire text at once, the model programmatically examines and decomposes the prompt, making recursive calls to itself over specific snippets. This "divide and conquer" approach allows the system to navigate and reason over massive datasets that would otherwise exceed its memory capacity.

**Key findings from the research include:**

*   **Massive Scalability:** RLMs successfully process inputs up to two orders of magnitude (100x) beyond the model’s native context window.
*   **Superior Performance:** Across four diverse long-context tasks, RLMs significantly outperformed both base LLMs and existing long-context scaffolding techniques. Notably, this performance advantage persisted even for shorter prompts that technically fit within standard context windows.
*   **Cost Efficiency:** Despite their recursive nature, RLMs maintain a cost per query that is comparable to, or in some cases cheaper than, traditional processing methods.

By shifting the focus from architectural scaling (increasing context windows) to inference-time scaling (recursive processing), RLMs provide a robust framework for handling extremely long-form data without sacrificing quality or increasing computational overhead.

该技术栈由四个核心组件组成：
*   **矢量切片：** 以 PMTiles 格式打包的本地地图数据。
*   **路径规划：** 基于 Valhalla 的 API，用于实现任意地点间的离线导航。
*   **地理编码：** 基于 Nominatim 数据构建的轻量级 SQLite 搜索引擎，支持正向和逆向地理编码。
*   **UI：** 连接到这些本地端点的 MapLibre 前端，用户也可以连接自定义的前端。

项目路线图包括：用于无缝地图更新的后台服务、对自定义 GeoJSON 叠加层（POI 和地理围栏）的支持，以及扩展对 K3s、Portainer 和 Mender 等平台的部署支持。
