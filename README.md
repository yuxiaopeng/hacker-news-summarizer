**原文标题**: Meta Platforms: Lobbying, dark money, and the App Store Accountability Act

**原文链接**: [https://github.com/upper-up/meta-lobbying-and-other-findings](https://github.com/upper-up/meta-lobbying-and-other-findings)

一项开源情报 (OSINT) 调查显示，Meta Platforms 策划了一场复杂的、多渠道的影响力行动，旨在推动通过《应用商店责任法案》(ASAA)。该立法旨在将年龄验证的监管和财务负担从社交媒体平台转移到苹果和谷歌的应用商店，从而使 Meta 有效规避新的合规要求。

**该行动的关键组成部分包括：**

*   **创纪录的支出：** 2025 年，Meta 在联邦游说上花费了创纪录的 2630 万美元，并在 45 个州部署了 86 名游说者。据报道，在路易斯安那州，Meta 的游说者起草了 HB-570 法案的初步文本，该法案已获得一致通过。
*   **“草根”伪装倡议：** Meta 秘密资助了“数字童年联盟”(DCA)，这是一个 501(c)(4) 类型的“草根”组织。DCA 提倡针对应用商店的儿童安全法律，却忽略了社交媒体平台。调查发现，DCA 缺乏标准的注册记录，并与美国国家性剥削中心 (NCOSE) 共享基础设施。
*   **超级政治行动委员会 (Super PACs)：** Meta 向加利福尼亚州、德克萨斯州等地的州级超级政治行动委员会投入了超过 7000 万美元，以影响立法优先事项，并强化与 ASAA 目标一致的家长监督论调。
*   **战略联系：** Meta 的游说工作与主要的“黑钱”网络相连。例如，Meta 的一名关键游说者还担任新风险基金 (New Venture Fund) 的董事会主席，该基金是阿拉贝拉顾问公司 (Arabella Advisors) 网络旗下的旗舰实体。
*   **立法成果：** 《应用商店责任法案》已在犹他州、路易斯安那州和德克萨斯州签署成为法律，大约 17 个其他州也引入了类似的法案。

调查结论指出，虽然 Meta 将其对 ASAA 的支持包装为一项儿童安全倡议，但其主要目标是迫使竞争对手承担监管成本，同时确保 Meta 的应用程序无需面对任何新的合规指令。

**原文标题**: TUI Studio – visual terminal UI design tool

**原文链接**: [https://tui.studio/](https://tui.studio/)

**TUI Studio** 是一款“类 Figma”的可视化编辑器，旨在简化终端用户界面（TUI）的创建。它允许开发者通过拖放界面构建终端应用程序，而无需手动编写复杂的布局代码。

**核心特性：**
*   **可视化画布：** 实时编辑器，支持 ANSI 实时预览，并拥有支持绝对定位（Absolute）、Flexbox 和网格（Grid）模式的布局引擎——类似于浏览器中的 CSS。
*   **丰富的组件库：** 包含 20 多个内置元素，如方框（Box）、按钮、表格、列表、加载动画（Spinner）和进度条。
*   **主题定制：** 支持八种流行配色方案，包括 Dracula、Nord 和 Gruvbox，可即时更新画布显示。
*   **文件管理：** 项目以可移植的 `.tui` JSON 文件形式保存，无需账号或云端连接，便于共享或提交至 Git。

**当前状态与框架支持：**
该工具目前处于 **Alpha** 测试阶段。虽然设计环境已具备功能，但**代码导出功能尚未启用**。待完全实现后，TUI Studio 计划为以下六大主流框架生成生产级代码：
*   **Ink** (TypeScript)
*   **BubbleTea** (Go)
*   **Blessed** (JavaScript)
*   **Textual** (Python)
*   **OpenTUI** (TypeScript)
*   **Tview** (Go)

**可用性：**
TUI Studio 适用于 macOS（针对 Apple 芯片已优化）、Windows 和 Docker。由于 Alpha 版本尚未进行代码签名，用户在安装过程中必须手动绕过安全警告（Gatekeeper/SmartScreen）。核心编辑器目前可免费使用，未来计划推出包含团队协作和云同步功能的“Pro”版本。

## 7. 存储桶抢注（终于）终结了

**原文标题**: Bucketsquatting is (finally) dead

**原文链接**: [https://onecloudplease.com/blog/bucketsquatting-is-finally-dead](https://onecloudplease.com/blog/bucketsquatting-is-finally-dead)

AWS has introduced a new naming convention to eliminate “bucketsquatting” (or “bucketsniping”), a long-standing security vulnerability where attackers register deleted or predictable S3 bucket names to intercept data. Because S3 bucket names are globally unique, attackers often guess and claim names like `myapp-us-east-1` once they are released or before they are created by the legitimate owner.

The solution is a new account-specific namespace syntax: 
**`<yourprefix>-<accountid>-<region>-an`**

The suffix **`-an`** (account namespace) ensures that only the specified AWS account ID can create buckets using that name. If a different account attempts to register the same name, AWS will return an `InvalidBucketNamespace` error. Furthermore, the bucket must be created in the region specified in the name, or it will be rejected.

**Key features and enforcement:**
*   **Mandatory Recommendation:** AWS now considers this the default standard for all new buckets.
*   **Policy Enforcement:** Security administrators can mandate this naming convention across an entire organization using Service Control Policies (SCPs) with the new condition key `s3:x-amz-bucket-namespace`.
*   **Migration Required:** The protection is not retroactive. Existing buckets remain vulnerable unless users migrate their data to new buckets following the namespace pattern.

Regarding other providers, Google Cloud Storage mitigates this via domain name verification, while Azure Blob Storage remains susceptible to similar risks, complicated by a restrictive 24-character limit on storage account names. 

Ultimately, the author concludes that while legacy buckets require manual migration, this new namespace effectively ends the threat of bucketsquatting for new AWS deployments.

## 8. 敢于出丑

**原文标题**: Willingness to look stupid

**原文链接**: [https://sharif.io/looking-stupid](https://sharif.io/looking-stupid)

在《甘愿显得愚蠢》一文中，沙里夫·沙米姆（Sharif Shameem）指出，害怕显得愚蠢是通往卓越创意的主要障碍。他观察到，随着创作者积累了经验或声望，往往会患上“诺贝尔奖综合征”：维持高标准的压力阻碍了他们去探索那些可能带来突破的“微小”或“古怪”的想法。

沙米姆提出了“阿迪勒定律”（Aadil’s Law），该定律认为：一个人最终产出的质量，与他在创作过程中愿意容忍的“愚蠢”或糟糕想法的数量成正比。他强调，创新往往源于年轻人或尚未证明自己的人，因为他们没有预期的负担，能够追随好奇心进入看似荒谬的领域。他以生物进化作类比，指出自然选择需要产生无数“不适宜”的突变，才能最终演化出像水母这样成功的生物。

作者识别了创作过程中的两种主要失败模式：
1. **过度分享：** 因不加选择地分享而显得愚蠢。
2. **分享不足：** 为了保护自尊心而避而不谈，导致作品平淡乏味。

沙米姆总结道，解决创作停滞的良方是将目标从“表现聪明”转变为“持续产出”。通过降低准入门槛并承担显得愚蠢的风险，创作者可以绕过脆弱的自我，重新找回发现伟大创意所需的勇气。最终，创作的成功与其说是天赋使然，不如说是取决于你今天是否愿意显得愚蠢。
