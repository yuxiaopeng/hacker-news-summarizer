## 7. 精灵的设计与实现

**原文标题**: Design and Implementation of Sprites

**原文链接**: [https://fly.io/blog/design-and-implementation/](https://fly.io/blog/design-and-implementation/)

Fly.io 推出了 Sprites，这是一种新型的“一次性”Linux 虚拟机，旨在实现近乎瞬时的创建和无缝的使用体验。Sprites 被描述为“圆珠笔式电脑”，提供 root 权限、100GB 持久化存储和自动休眠功能，使其运行起来如同高速、持久的 SSH 环境，且在闲置时几乎不产生费用。

为了实现瞬时创建和高扩展性，Fly.io 通过三个关键设计决策，脱离了其旗舰产品 Fly Machines 的架构：

1. **取消自定义容器镜像：** 与传统 Fly Machines 需要拉取并解压沉重的 OCI 镜像不同，所有 Sprites 都从标准化的预缓冲容器启动。这使创建时间从一分钟以上缩短至仅需一两秒。
2. **基于对象存储的磁盘：** Sprites 不再通过挂载 NVMe 驱动器将虚拟机绑定到特定物理硬件，而是将兼容 S3 的对象存储作为数据源。通过使用修改后的 JuiceFS 和 Litestream 技术栈，数据在本地进行缓存以保证性能，同时保持可移植性。这实现了服务器间的轻松迁移以及极速的检查点/恢复功能。
3. **由内而外的编排：** 为简化管理，编排逻辑（如存储处理和日志记录）被移至虚拟机的根命名空间内部。用户代码在嵌套容器中运行，使得 Fly.io 可以在不中断主机或全局状态的情况下更新平台。

Sprites 针对交互式任务、原型开发和 AI 驱动开发进行了优化，并预装了 Claude 和 Gemini 等代理。虽然生产规模的应用和数据库仍推荐使用 Fly Machines，但 Sprites 提供了一个灵活的环境，用户可以为即时的临时任务快速生成数十台独立的计算机，而无需承担传统云资源配置的开销。
