**原文标题**: Zed Is Our Office

**原文链接**: [https://zed.dev/blog/zed-is-our-office](https://zed.dev/blog/zed-is-our-office)

Zed Industries 将其自身的文本编辑器 Zed 作为其虚拟办公室，突出了其协作功能和无缝集成。文章强调，协作不仅仅是一个附加功能，而是融入了 Zed 的 DNA，使用 CRDT 实现无冲突编辑和低延迟，无论位置或团队规模如何。与其他需要扩展和链接共享的编辑器不同，Zed 使用 GitHub 句柄进行轻松设置，并包含内置的音频和屏幕共享功能。

Zed 旨在促进持续的对话，并将讨论、编辑和见解与代码集成。虽然开发工作因用户要求的功能（如调试、窗口和 git 支持）而优先处理并有所偏离，但他们的主要目标仍然是构建最强大的多人软件开发工具。协作功能目前处于 Alpha 阶段，可免费使用。文章最后邀请用户下载 Zed 并考虑加入团队。

## 4. The Monks in the Casino

**原文标题**: The Monks in the Casino

**原文链接**: [https://www.derekthompson.org/p/the-monks-in-the-casino](https://www.derekthompson.org/p/the-monks-in-the-casino)

德里克·汤普森的《赌场里的僧侣》探讨了年轻男性逐渐退出社会交往、转向色情和网络赌博等孤独活动的趋势，形成了一种现代形式的禁欲主义。他以性瘾者“斯皮沙克”和沉迷体育博彩的“凯尔”为例，来说明这种现象。

汤普森认为，这不仅仅是“孤独危机”，而是“缺乏孤独危机”，即孤独、基于屏幕的活动的诱惑超过了对人际关系的渴望。这得益于一种越来越重视孤独的文化和迎合它的经济。

他批评了向“赌场资本主义”的转变，在这种转变中，在线赌博和金融投机的便利，加上难以实现传统成功标志（如拥有住房）的困难，将年轻男性推向了冒险、快速致富的计划。偏向数字空间而非实体社区建设的政策选择加剧了这种情况。

文章将这种现代“僧侣”与马克斯·韦伯描述的新教伦理进行了对比。这些现代僧侣在社交领域规避风险，但在金融领域却追逐风险，而不是在社会上慷慨大方，在经济上谨慎。汤普森总结说，社会需要重新评估其优先事项，培养社区和现实世界的参与，而不是优先考虑轻松访问在线赌场。

## 5. Checkout.com遭黑客攻击，拒绝支付赎金，捐款给安全实验室

**原文标题**: Checkout.com hacked, refuses ransom payment, donates to security labs

**原文链接**: [https://www.checkout.com/blog/protecting-our-merchants-standing-up-to-extortion](https://www.checkout.com/blog/protecting-our-merchants-standing-up-to-extortion)

支付处理平台Checkout.com遭受“ShinyHunters”犯罪团伙攻击，该团伙声称窃取了数据并勒索赎金。该公司已确认在2020年及之前使用的旧版第三方云文件存储系统遭到未经授权的访问，影响了不到25%的现有商户。该系统包含内部运营文件和商户入驻材料。

Blender基金会推出Blender实验室，致力于探索具有挑战性的前瞻性项目，以保持Blender的活力。此举旨在解决Blender因其成熟度、技术债务和对稳定性的重视而导致添加新功能日益复杂的问题。

Blender实验室允许设计师、开发者和研究人员在正常的发布周期之外进行实验和“破坏”。实验室项目是公开的，并在blender.org/lab上可见，详细说明目标、时间表、参与者，并提供中间版本以供测试和反馈。

实验室活动可以是：

*   **应用研究：** 侧重于基于当前研究开发突破性解决方案。
*   **学术研究：** 涉及Blender开发者为大学和研究中心项目的技术实施提供建议。

初始项目包括对触摸/笔界面、VR/XR、体积渲染和光传输的探索。未来的项目可能涉及USD编写和AI/ML集成。

要为实验室提议项目，感兴趣的各方应向Blender基金会提交详细的公开提案。项目的采纳取决于资金、与Blender使命的相关性以及申请人的经验等因素。

Ton Roosendaal自2018年以来一直倡导这个项目。它得益于不断增长的社区和企业支持，并将需要通过Blender开发基金持续资助。

该帖子强调了验证在防止诈骗中的重要性，尤其是在快速数字化的地区，并举例说明了诈骗者使用社会工程学来诱骗用户侧载恶意软件。验证被视为一种通过迫使攻击者使用可追溯的身份来运作，从而改变不断打击恶意应用程序的“打地鼠”动态的方式。

目前，向Google Play之外分发应用程序的开发者可以使用早期访问计划，Play开发者很快也会跟进。Google鼓励开发者提供关于简化验证体验的反馈。相关资源包括视频演示和常见问题解答。

## 10. Kratos - 云原生 Auth0 开源替代方案（自托管）

**原文标题**: Kratos - Cloud native Auth0 open-source alternative (self-hosted)

**原文链接**: [https://github.com/ory/kratos](https://github.com/ory/kratos)

Ory Kratos：面向云原生应用设计的开源、API优先的身份和用户管理系统。它提供了一个可自托管的Auth0及类似服务的替代方案，集中管理登录、注册、账户恢复、验证和个人资料管理等关键工作流程，从而将身份验证逻辑从应用程序代码中移除。

主要功能包括自助服务登录/注册、MFA、个人资料管理、身份验证模式和管理API。Ory Kratos旨在扩展、与其他Ory工具（Hydra、Oathkeeper、Keto）集成，并适应Kubernetes环境。

您可以将Ory Kratos部署为Ory Network上的托管服务，或选择自托管。Ory Network提供完全托管的解决方案，具有诸如密钥支持、预构建的UI组件、OAuth2/OIDC和符合GDPR的存储等功能。自托管提供完全控制权，但需要管理基础设施。对于业务关键型系统，Ory Enterprise License (OEL) 提供额外功能（SCIM、SAML、CAPTCHA），带有 SLA 的安全版本、高级支持以及对私有 Docker 注册表的访问。

本文档提供了使用 Ory CLI 创建新项目的快速入门指南，并重点介绍了使用 Ory Kratos 的公司。
