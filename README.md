## 6. Show HN: 我的基于 Postgres 的私有 GitHub

**原文标题**: Show HN: My Private GitHub on Postgres

**原文链接**: [https://github.com/calebwin/gitgres](https://github.com/calebwin/gitgres)

**GitGres** 是一个开源的 GitHub 私有重新实现版本，旨在完全运行在 Postgres 后端。该项目基于 Rust 开发，它将包括 Git 对象、包文件（packfiles）、PR、议题（issues）和评论（reviews）在内的所有组件均作为行存储在 Postgres 中，而非传统的服务器文件系统。

该项目被定位为团队和智能体（agents）的“起点”，适用于那些需要比云端托管 GitHub 提供更多基础设施控制权的场景。它解决了三个主要的权衡问题：
*   **可用性与成本：** 用户可以利用 Postgres 的分层存储在保持高可用性的同时管理成本。
*   **延迟与成本：** 可以通过标准的 Postgres 缓存机制来优化性能。
*   **吞吐量与一致性：** GitGres 支持可调节的一致性和吞吐量，是本地部署或同地协作智能体环境的理想选择。

**核心功能与用法：**
GitGres 兼容标准 Git（Smart HTTP v1/v2），并完全支持官方的 **`gh` 命令行工具**。这使得用户可以像使用 GitHub 一样，通过命令行管理仓库、议题、发布版本和 PR。它还包含一个用于直接与数据库交互的自定义远程辅助程序（remote helper）。

**安装与局限性：**
安装需要 Rust 环境和可访问的 Postgres 实例。在构建二进制文件后，用户需要初始化模式（schema），并使用引导令牌（bootstrap token）启动服务器以获取管理权限。

目前，GitGres 存在一些局限性：它不支持 Web 界面、SSH 传输、GitHub Actions/工作流、Webhooks 或搜索功能。它主要面向需要高度可定制、数据库驱动，且针对程序化或智能体导向工作流进行优化的 Git 基础设施的用户。

## 7. 在浏览器中运行 Adobe 1991 年的 PostScript 解释器

**原文标题**: Running Adobe's 1991 PostScript Interpreter in the Browser

**原文链接**: [https://www.pagetable.com/?p=1854](https://www.pagetable.com/?p=1854)

The article describes **retro-ps**, a project that emulates Adobe’s 1991 PostScript Level 2 interpreter to run in modern web browsers and command-line environments. 

Originally found on the **HP C2089A PostScript Cartridge Plus** for the LaserJet III, this 2 MB ROM contains Adobe’s own reference implementation (version 2010.118). Despite being over 30 years old, the code remains a highly accurate renderer because the PostScript language has not changed significantly.

**Technical Implementation**
The emulator works by:
*   **CPU Emulation:** It simulates a Motorola 68020 CPU (an upgrade from the original 68000) to provide 16 MB of RAM, allowing for high-DPI rendering without memory errors.
*   **Hardware Faking:** Since the project lacks the original LaserJet mainboard ROM, it "fakes" the necessary hardware traps, such as printer-model IDs, engine-status polling, and interrupts, to satisfy the cartridge’s state machine.
*   **Lifting Constraints:** While the original hardware was limited to 300 DPI and fixed margins, `retro-ps` allows for custom paper sizes and resolutions up to ~1450 DPI. It also injects a custom "prolog" to scale halftone cells for higher resolutions.

**Functionality and Future Work**
The browser-based version operates entirely client-side via a drag-and-drop interface, requiring no server-side processing. The author notes that this "old code" is not merely "retro" but remains a productive tool. Future goals for the project include emulating other cartridge ROMs (like Pacific Page P·E) and porting the system to support the Intel i960-based architectures found in later LaserJet models.
